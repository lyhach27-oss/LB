게임 만들기 시작 전 규칙

AI한테 게임 만들어달라고 하기 전에, 아래 내용을 프롬프트에 꼭 넣어주세요.

①
반드시 지켜야 할 규칙 필수
1
숫자·색상·텍스트는 코드 안에 쓰지 말 것
크기, 색상, 문구 같은 값을 코드 안에 직접 쓰면 나중에 수정이 너무 어려워져요. 반드시 config.json 파일을 따로 만들어서 거기서 불러오게 해야 해요.
프롬프트에 이렇게 써요
숫자, 색상, 이미지 경로, 텍스트는 절대 코드 안에 하드코딩하지 말고 config.json 파일을 별도로 만들어서 거기서 불러와줘.
2
config.json에 반드시 있어야 하는 항목들
게임을 만들면서 AI가 config.json을 만들 때 아래 항목들이 포함되어 있는지 확인하세요.
instanceId
게임 고유 이름 (예: "game01") — 다른 게임과 겹치지 않게
canvasW / canvasH
화면 가로·세로 크기 (예: 500 / 900)
board.rows / cols
퍼즐 칸 수 (예: 8행 10열)
board.colors
사용할 색상 목록 (예: ["red","blue","green"])
urls.storeLink
다운로드 버튼 클릭 시 이동할 링크
3
화면 크기를 config에서 받아오게 할 것
물리 엔진이나 캔버스 크기를 500, 900처럼 숫자로 쓰지 말고, config.json의 canvasW와 canvasH를 읽어서 쓰게 해야 해요. 단, 값 할당은 반드시 초기화 함수 안에서 해야 해요.
프롬프트에 이렇게 써요
캔버스 가로·세로 크기는 코드에 직접 쓰지 말고, config.json의 canvasW와 canvasH 값을 읽어서 사용해줘. 단, 값 할당은 반드시 초기화 함수(init 함수) 내부에서 해줘.
4
모든 HTML id 앞에 게임 이름 붙이기
나중에 여러 게임을 한 화면에 합칠 때 이름이 겹치면 오류가 나요. config.json의 instanceId 값을 모든 id 앞에 붙여야 해요.
프롬프트에 이렇게 써요
HTML의 모든 id 값 앞에 config.json의 instanceId 값을 붙여줘. (예: id="game01-board") JS에서 getElementById를 쓸 때도 동일하게 적용해줘.
②
게임 시작할 때 쓰는 프롬프트 템플릿 복붙용
아래 내용을 게임 설명 앞에 붙여서 AI에게 보내세요.
게임 시작 공통 프롬프트 (앞에 붙이기)
아래 규칙을 반드시 지켜서 만들어줘.

1. 숫자, 색상, 이미지 경로, 텍스트는 코드에 직접 쓰지 말고 config.json 파일을 별도로 만들어서 거기서 불러와줘.
2. config.json에 instanceId, canvasW, canvasH, board.rows, board.cols, board.colors, urls.storeLink 항목을 포함해줘.
3. 캔버스 크기는 초기화 함수 내부에서 config.json의 canvasW, canvasH를 읽어서 할당해줘.
4. HTML의 모든 id 앞에 config의 instanceId 값을 붙여줘. JS의 getElementById도 동일하게 적용해줘.

⚠️ AI가 규칙을 안 지켰을 때
코드를 받고 나서 config.json이 따로 안 만들어졌거나, 코드 안에 숫자가 직접 쓰여 있다면 이렇게 말해주세요. "config.json이 별도 파일로 분리되어 있지 않아. 위의 규칙대로 다시 만들어줘. 기존 게임 로직은 건드리지 말 것."
게임 완성 후엔 라이브러리 등록 체크리스트 문서로 최종 확인해주세요.
이 규칙들을 처음부터 지키면 체크리스트에서 수정할 게 거의 없어요.
