# 팀 랜덤 선택기 웹앱 (Team Random Selector) 요구 사항 명세서 (PRD)

## 1. 개요 (Overview)
본 문서는 팀 내에서 역할(예: 발표 순서 지정, 당번 정하기)이나 항목(예: 회식 메뉴, 점심 메뉴)을 무작위로 추첨할 때 간단히 사용할 수 있는 '팀 랜덤 선택기' 웹 애플리케이션의 요구사항을 정의합니다. 

## 2. 목적 (Purpose)
- 팀원들이 빠르고 공정하게 무작위 선택(Random Selection)을 진행할 수 있도록 지원합니다.
- 복잡한 가입 절차 없이 URL 파라미터나 간단한 텍스트 입력을 통해 손쉽게 사용할 수 있습니다.
- 직관적이고 모던한 UI/UX를 제공하여 사용자 경험(UX)을 극대화합니다.

## 3. 기술 스택 (Tech Stack)
- **프론트엔드 프레임워크**: Next.js (최신 App Router 활용 권장)
- **스타일링**: Tailwind CSS (빠르고 일관성 있는 디자인 시스템 구축)
- **배포 및 인프라**: Cloudflare Pages (빠른 정적 호스팅 및 에지 컴퓨팅)
- **애니메이션**: CSS 애니메이션 및 필요시 Framer Motion (부드러운 시각 효과 추가)

## 4. 핵심 기능 요구 사항 (Core Features)

### 4.1. 항목(이름) 입력 및 관리
- **텍스트 기반 입력**: 사용자가 웹 사이트 화면 내 텍스트 영역(Textarea)이나 인풋 필드를 통해 쉼표(,)나 줄바꿈(Enter)으로 구분하여 추첨 대상(이름, 메뉴 등) 목록을 직접 입력할 수 있어야 합니다.
- **URL 연동 (필수 기능)**: 
  - `/?names=A,B,C` 와 같이 URL 쿼리 파라미터를 통해 추첨 목록 데이터가 자동으로 화면에 렌더링 되어야 합니다.
  - 사용자가 현재 구성한 항목들을 다른 팀원에게 공유할 수 있는 **'공유 링크 복사'** 기능이 있어야 합니다.

### 4.2. 무작위 추첨 로직
- "1명 뽑기", "순서 섞기" 등의 기본적인 추첨 기능이 동작해야 합니다. (초기 V1.0은 단순 랜덤 선택에 집중)
- 추첨 후 재추첨 시, 이미 뽑힌 사람/항목을 "제외하고 뽑기" 또는 "포함해서 다시 뽑기" 옵션을 직관적으로 제공하면 좋습니다.

### 4.3. UI/UX 및 디자인 가이드라인
- **모던하고 심플한 디자인**: 여백을 충분히 살린 미니멀 디자인. 텍스트 가독성을 최우선으로 고려합니다.
- **다크/라이트 모드 지원 (선택/권장)**: 환경에 맞춰 쾌적하게 볼 수 있는 테마 지원.
- **다이내믹 애니메이션 효과**: 
  - 버튼을 클릭할 때의 Hover/Active 효과.
  - 추첨이 '진행 중'일 때 긴장감을 주는 모션(예: 슬롯머신처럼 이름이 빠르게 바뀌는 슬라이드 효과).
  - 결과창이 등장할 때의 강조 효과(예: 폭죽(Confetti) 효과 또는 부드러운 페이드인 모션).
- **반응형 웹**: 모바일 기기(iPhone, Android 등)에서 접속해도 UI가 깨지지 않고 자연스럽게 사용할 수 있어야 합니다.

## 5. 비기능 Requirements 및 배포 계획
- **정적 배포 호환성**: Next.js의 기능을 활용하되 `output: 'export'` 등 설정으로 정적 사이트(Static Site) 생성이 가능하게 구현하여 Cloudflare Pages에서 빠르고 저렴하게 서비스될 수 있도록 합니다. (필요 시 API 라우트를 위해 Edge Runtime 도입 가능)
- **SEO 및 성능**: 즉각적인 반응을 보여주도록 최적화하며, 접근성을 함께 고려합니다. 

## 6. 마일스톤 (Milestone)
1. **Phase 1**: 프로젝트 세팅 (Next.js + Tailwind CSS) 및 `/docs` 라우트에 설계/명세서 정리.
2. **Phase 2**: 메인 화면 퍼블리싱 (텍스트 입력 폼, 항목 리스트업 UI 구현).
3. **Phase 3**: URL 쿼리 스트링(`?names=...`) 연동 및 '클립보드에 링크 복사' 기능 추가.
4. **Phase 4**: 랜덤 추첨 알고리즘 및 시각적 애니메이션 (결과 효과 포함) 결합.
5. **Phase 5**: Cloudflare Pages 레포지토리 연동 및 자동 배포 시스템 구축 / QA.
